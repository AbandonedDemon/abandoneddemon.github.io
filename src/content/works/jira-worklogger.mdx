## Overview

{/* TODO(author): one or two paragraphs on what the tool is and who uses it. */}

A time-logging dashboard and Chrome extension that streamlines logging work against Jira issues,
built for developers who would otherwise skip it.

## Problem

{/* TODO(author): what was painful before this existed. */}

Logging time issue-by-issue in Jira is slow and easy to forget, so worklogs end up incomplete and
sprint reporting suffers.

## Solution

{/* TODO(author): the system you built, at a high level. */}

A dashboard with a bulk-logging workflow plus a Manifest V3 Chrome extension that surfaces logging
right where the work happens.

## My Contribution

{/* TODO(author): what you personally designed and built. */}

- Designed and built the time-logging dashboard
- Implemented the bulk-logging workflow
- Built the Manifest V3 Chrome extension
- Added sprint visibility and worklog export

## Technical Highlights

- Time-logging dashboard with sprint visibility
- Bulk worklog entry across multiple issues
- Chrome Extension on Manifest V3
- Jira Cloud and Server (Data Center) compatibility
- Worklog export functionality
